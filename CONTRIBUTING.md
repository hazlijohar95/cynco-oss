# Contributing

## Fresh clone setup

1. Install [proto](https://moonrepo.dev/docs/proto):

   ```bash
   curl -fsSL https://moonrepo.dev/install/proto.sh | bash
   ```

2. From the repo root, install the pinned toolchain and dependencies:

   ```bash
   proto use
   pnpm install
   ```

3. Verify everything is green:

   ```bash
   moon run :build :test :typecheck root:lint root:lint-css
   ```

## Day-to-day

- `moonx demo:dev` — Vite kitchen-sink playground on port 4600.
- `moonx docs:dev` — docs site on port 4700.
- `moonx <project>:test` / `:typecheck` — focused verification.
- `moon run root:format root:lint` — repo-wide format and type-aware lint.
- `moonx ledger-store:benchmark` / `accounts:benchmark` / `journals:benchmark` —
  mitata benchmarks.

Git hooks are generated by moon on any moon command (`vcs.sync`): pre-commit
runs affected typechecks, the repo formatter, and lint-staged.

## Parallel worktrees

Multiple checkouts (humans or agents) can run dev servers and e2e suites at the
same time without port collisions. `scripts/wt.ts` manages sibling git worktrees
at `<primary>-worktrees/<name>/`, each with its own port block:

```bash
bun scripts/wt.ts create <branch>   # add worktree, claim ports, pnpm install
bun scripts/wt.ts list              # every worktree: name, offset, branch, path
bun scripts/wt.ts sync              # idempotently (re)write .env.worktree files
bun scripts/wt.ts remove <name>     # kill its servers, remove worktree + merged branch
```

(also available as `moonx root:wt -- <subcommand>`.)

Each worktree owns a gitignored `.env.worktree` at its root with
`CYNCO_PORT_OFFSET` — a stable slot index × 1000 (the primary checkout is slot 0
and keeps the historical ports). Port-binding moon tasks load the file via
`envFile: '/.env.worktree'` (silently absent in the primary checkout) and bind
`base + offset`: demo 4600, docs 4700, journals e2e 4283, accounts e2e 4383.
Playwright configs additionally load the file themselves through
`scripts/load-worktree-env.mjs`, so direct `pnpm exec playwright test` runs
shift too. `git worktree` surgery done by hand is repaired by `sync`, which
keeps valid claims and assigns the lowest free slot to newcomers.

## Rules

- Dependencies come from the `catalog:` in `pnpm-workspace.yaml`; internal
  packages use `workspace:*`.
- Amounts are integer minor units end to end; no floats touch money.
- Component CSS uses data-attribute selectors and
  `--*-override → --*-theme-* → default` variable chains only.
- Commit messages: `<type>(<scope>): <subject>`, 72-column wrap.
